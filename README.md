# 素人种草运营工作台

本地网页工作台，用来管理 40+ 兼职/账号的每日内容派发、素材调度、爆款提权、图文/视频交付、账号数据监控和爆款提醒。

## 启动

双击根目录的 `启动素人工作台.command`，或在终端运行：

```bash
cd /Users/licc/Desktop/素人系统/app
npm install
npm run start
```

默认地址：

```text
http://127.0.0.1:5174
```

同一局域网里的工作人员要访问时，在主机 `app/.env` 设置：

```env
SOUREN_HOST=0.0.0.0
SOUREN_ACCESS_CODE=给工作人员的访问码
```

重启后进入「运行状态」查看局域网网址，发给工作人员即可。工作人员只打开网页、输入访问码、查看今日任务、复制交付内容、下载素材和标记状态；图片生成、爆款分析和系统数据仍在主机本地完成。系统不提供本地目录直达入口，素材目录只作为后台保存和读取位置。
如果设置 `SOUREN_HOST=0.0.0.0` 但没有填写 `SOUREN_ACCESS_CODE`，系统会拒绝启动，避免局域网裸奔。

## 桌面客户端

开发机上可以直接用桌面窗口运行：

```bash
cd /Users/licc/Desktop/素人系统/app
npm run client
```

生成 Mac 桌面应用目录：

```bash
cd /Users/licc/Desktop/素人系统/app
npm run client:verify
```

输出目录是 `app/release/`。客户端包不会带真实 `.env`、数据库或素材；运行时会在本机自己的工作数据目录里创建数据和素材库。Mac `.app` 会用 Electron 内置 Node 启动本地后端并打开工作台窗口，不需要工作人员手动启动服务。`client:verify` 会先生成 `.app`，再实际启动打包后的客户端，确认内置后端和独立工作数据目录可用。

## 数据和素材

- 结构化数据：`/Users/licc/Desktop/素人系统/data/souren.sqlite`
- 真实案例素材：`/Users/licc/Desktop/素人系统/素材库/真实案例`
- 通用素材库：`/Users/licc/Desktop/素人系统/素材库/通用素材`，默认包含 `医院素材`、`套图素材`、`备用素材`。
- 网页端只保存路径和状态，不把图片、视频写进数据库。
- 新建案例后，系统会自动创建 `00-原始素材`、`01-已筛选素材`、`02-生成补充`、`03-交付给兼职`。
- 新建和编辑案例只让工作人员填写兼职微信昵称、抖音主页/作品链接、项目和共享原始素材路径；系统自动生成近期排期，内部写作参考由后台处理。
- 共享盘/服务器目录是工作人员放素材的入口；新建案例会先校验这个目录并自动同步一次到本地受控案例目录，后续目录新增文件时再通过「同步共享素材」补同步。系统会记录原始来源路径，后续生成和交付优先使用本地副本。
- 系统会按目录名和文件名自动给素材分类：案例人物、案例过程、案例对比、案例场景、医院素材、套图素材、生活场景、文字卡等。分类不准时可在素材清单里手动改用途。

## 工作人员每天只做两件事

1. 打开「今日中控台」，按「今日操作队列」逐条处理：生成候选、选择、生成交付、补素材、剪辑、查看交付内容，然后在网页内复制文案、下载图片/视频，用电脑微信发送。
2. 打开「爆款链接」，把看到的抖音爆款作品链接贴进系统；链接进入待分析状态，主机侧写回分析结果后再生成同类型内容。

完整每日操作步骤见：`日常运营SOP.md`。

## 核心模块

- 今日中控台：首屏只保留一张「今日操作队列」；后续账号预览默认折叠，按操作队列聚合素材同步、剪辑、爆款互动和交付，同一条任务只在队列里出现一次，工作人员按队列处理即可。
- 运营建议：异常账号和账号策略动作只给工作人员需要处理的下一步，例如补排期、处理素材缺口、加爆款提权、安排爆款互动，或一键生成偏冷补内容/增长承接排期。
- 排期规划：按未来 7/14/30/60 天查看所有账号内容安排；生成、交付和派发必须等任务排到今日操作队列队首后处理。已派发任务不再挡队列，收到兼职完成回复后从「等待兼职确认」里标记完成。
- 案例库：新建、批量导入、搜索筛选、扫描服务器案例库和删除案例；批量导入会先整批预检共享目录，通过后自动同步素材和生成排期，避免半导入；服务器案例库登记使用网页弹窗绑定微信、抖音、项目和共享素材路径，删除进入工作台内确认弹窗，确认后同步删除本地素材副本。
- 新建和批量导入案例必须填写兼职微信昵称、抖音主页/作品链接、项目和共享原始素材路径；系统从源头避免创建无法采集、无法同步素材或项目不清楚的空账号。
- 补登记：历史遗留账号如果缺微信昵称、抖音链接、项目或共享原始素材路径，会进入首页「补登记」队列，排到队首后打开案例补齐资料。
- 共享素材：案例素材按账号单独同步；医院素材、套图素材等通用素材放入通用素材库，运行状态页可扫描、预览并修正分类/用途/状态，日常养号和爆款提权可复用通用素材。
- 素材同步提醒：首页会检测案例共享目录里新增但尚未同步的素材，显示「待同步共享素材」，可一键同步到本地案例目录。
- 爆款链接：工作人员只贴链接且链接必填；待分析链接不进入今日操作队列，主机侧通过 `/api/viral-templates/:id/analysis-result` 写回内容提取和结构分析后，只给可投放账号批量生成候选。
- 网页交付面板：交付内容直接在网页呈现，系统后台只保留必要文本和素材副本；弹窗顶部会锁定当前收件微信，工作人员必须先确认收件微信，再按当前步骤依次复制文案、下载图片和视频；发布要求已写进「发给兼职文案」，不再额外制造复制步骤；标记已派发后进入「等待兼职确认」，文案和素材下载都变为只读，避免错发或重复发送。
- 今日操作队列只完整显示第一条任务；后面的任务只汇总数量和类型，第一条处理完才继续下一条；发完微信后从今日队列消失，清空后显示「今天发完了」。已经发出但等对方回复的内容单独放在折叠的「等待兼职确认」，不阻塞当天队列。
- 后续账号预览和异常账号只做概览；不是当前队首账号时不能直接打开详情，避免新手跳号处理。
- 不提供一键批量准备入口；后台准备、生成候选和生成交付接口也只处理当前队首，避免一次生成多条待发送内容。
- 派发确认：列表页只允许打开交付内容；只有交付弹窗里能点「已用微信发送」，避免没看收件人和发送步骤就误标派发。
- 队首防呆：排期规划和案例详情可以查看任务；不是队首的任务不提供复制、派发、生成候选、选择候选、生成交付、素材同步/扫描/标记、图片任务、剪辑完成、爆款互动等执行按钮。已派发任务只在「等待兼职确认」里做完成确认，不允许重新复制下载。后端接口也会拒绝普通调用绕过队首推进任务。
- 交付锁定：内容进入交付链路后，只能查看和确认记录，不能重新生成交付或改选候选，避免已派发任务被打回。
- 失联账号：已派发超过 3 天仍未确认完成时，系统自动暂停这个账号，撤下普通队列、剪辑和采集任务；首页不再为失联账号生成新的待办。
- 恢复账号：兼职回来后在案例详情点「恢复账号」，系统会把陈旧派发任务标为「已取消」，并补新的近期排期。
- 视频/口播交付会额外展示口播字幕、固定剪辑配方和源素材下载；剪辑任务必须先打开完整剪辑要求并确认看完固定配方，剪辑或发布完成后只在系统标记完成，不要求上传成片。
- 剪辑任务防重：同一条排期只会有一个剪辑任务，重复点击会回到已有任务，不会制造重复待办。
- 图片任务：只从素材缺口或明确排期用途创建。接口未接入时显示“待接入图片接口”，仍可生成本地图片任务说明和网页提示词；接口接入后可在网页内生成并下载图片。
- 合成图参考：图片任务会自动带上“案例人物/变化参考”和“医院/通用场景素材”，方便后续用 Image2 做人物与医院场景、套图结构的组合。
- 剪辑任务：生成固定剪辑配方，排到队首后在网页内打开完整要求并确认看完配方，再标记完成；不要求打开本地目录。
- 抖音账号监控：新建案例填写抖音主页后，系统按账号活跃度维护采集任务；正在投放或起量的账号勤采，偏冷账号降频，休眠账号一周一次。采集清单按待首采、起量、活跃、偏冷、休眠排序，不按账号数硬跑。主机端可以运行 `npm run collect:douyin -- --register --limit 10`，用已登录抖音的 Chrome 打开到期主页，解析账号和作品数据后写入后台；解析不到真实数据时会跳过，不伪造采集结果。
- 系统后台：`/api/agent-work` 聚合待分析爆款、到期 Chrome 采集和图片任务，只在「运行状态」里折叠展示，不进入今日首页队列。
- 任务减量：系统会根据采集数据调整排期，连续低播放账号自动改为两天一条或暂停自动补新排期；已生成候选、已锁定或可交付但还没派发的旧任务会同步撤下，只有起量账号继续保持高频承接。
- 固定口径：案例会生成 `00-兼职须知.txt`；交付弹窗只在首次交付时显示可复制的兼职须知，后续只保留已发过提示。每次交付的发给兼职文案内置固定发布说明，包含时间窗、封面、话题、素材顺序和完成回复口径，减少每天重复解释。
- 账号策略动作：系统把采集后的账号表现转成工作人员需要处理的操作队列，包括爆款互动、偏冷补内容和增长承接；偏冷和增长动作只会按账号投放频率找下一个空档生成，不给休眠、暂停或当天已有排期的账号额外加任务。
- 爆款提醒：作品数据达到阈值后，首页提醒工作人员安排助理号/阑尾号去评论区互动。

## 自动化边界

- 系统可以独立完成：排期、候选稿、交付内容、素材路径管理、图片提示词、剪辑任务、账号采集任务登记、Chrome 采集执行、爆款提醒记录、爆款分析结果写回和后台动作聚合。
- 系统可以接入图片接口后完成：图片生成任务。文案、排期、交付和账号监控不依赖图片接口。
- 抖音数据不要求工作人员手动录入：工作人员只在新建案例时填写抖音主页；主机端用已登录抖音的 Chrome 查看主页，并把账号和作品数据写入后台。macOS 主机可直接在 `app/` 下运行 `npm run collect:douyin -- --register --limit 10`；先测试解析可用 `npm run collect:douyin -- --dry-run --limit 3`。
- 如果当天还没采集，状态只放在「运行状态」的折叠「系统后台」，不进入首页首屏和异常账号；系统不会伪造播放量或粉丝量。
- 不依赖当前聊天窗口时，客户端仍能运行；但“替你看抖音、临时判断、跨网页操作”这类能力需要浏览器协同或人工完成。

## 环境变量

在 `app/.env` 里预留：

```env
PORT=5174
SOUREN_HOST=127.0.0.1
SOUREN_ACCESS_CODE=
SOUREN_CASE_LIBRARY_ROOT=
SOUREN_SHARED_MATERIAL_ROOT=
DOUYIN_COLLECTION_CHECK_INTERVAL_MS=21600000
IMAGE_API_KEY=
IMAGE_API_URL=
IMAGE_MODEL=gpt-image-1
IMAGE_SIZE=1024x1024
```

图片生成只需要填写 `IMAGE_API_KEY`；`IMAGE_API_URL` 只是接其他供应商或本地服务时的可选覆盖。不填密钥时，图片任务仍会保留提示词，不影响文案、排期、交付和账号监控。

当前图片接口未接入时，文案、排期、网页交付和账号监控流程仍可正常使用。抖音数据默认走 Chrome 登录态采集任务，由主机端采集后写入后台。

`DOUYIN_COLLECTION_CHECK_INTERVAL_MS` 只控制后台多久检查一次到期账号；实际某个账号是否进入采集清单，由账号活跃度策略决定。已经等待 Chrome 的账号不会重复排队。

Chrome 采集执行器只在主机侧运行，不出现在工作人员首页。首次运行前确保 Google Chrome 已登录抖音，并在 Chrome 里允许 Apple Events 执行 JavaScript；如果抖音页面结构变化导致解析不到粉丝或互动数据，脚本会输出 `skipped`，保留等待采集状态，不会把空数据写回系统。

如果待分配案例集中放在共享盘或服务器挂载目录，可把 `SOUREN_CASE_LIBRARY_ROOT` 设置为那个“待分配案例根目录”，案例库页点击“扫描服务器案例库”后可一键登记。登记后系统会把该目录绑定为案例的共享原始素材路径，并同步进本地案例目录。

如果通用素材库放在共享盘或服务器挂载目录，可把 `SOUREN_SHARED_MATERIAL_ROOT` 设置为那个绝对路径。两者分工不同：`SOUREN_CASE_LIBRARY_ROOT` 放某一个人/某一个案例的素材，`SOUREN_SHARED_MATERIAL_ROOT` 放医院环境、套图模板、生活场景这类所有账号都能复用的素材。
