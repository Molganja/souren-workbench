# 素人种草运营工作台

本地网页工作台，用来管理 40+ 兼职/账号的每日内容派发、素材调度、爆款提权、图文/视频交付、抖音核对和数据回填。

## 启动

双击根目录的 `启动素人工作台.command`，或在终端运行：

```bash
cd /Users/licc/Desktop/素人系统/app
npm install
npm run start
```

默认地址：

```text
http://127.0.0.1:5174
```

## 数据和素材

- 结构化数据：`/Users/licc/Desktop/素人系统/data/souren.sqlite`
- 真实案例素材：`/Users/licc/Desktop/素人系统/素材库/真实案例`
- 网页端只保存路径和状态，不把图片、视频写进数据库。
- 新建案例后，系统会自动创建 `00-原始素材`、`01-已筛选素材`、`02-生成补充`、`03-交付给兼职`、`04-发布回收`。
- 新建案例默认随机生成人设，并自动生成近期排期；工作人员只需要填对接咨询/负责人和抖音主页/作品链接，其他信息可后续编辑。

## 工作人员每天只做两件事

1. 打开「今日中控台」，看今天每个兼职该发什么，生成候选、选择、复制文案、打开交付文件夹，然后用电脑微信发送。
2. 打开「爆款链接」，把看到的抖音爆款作品链接贴进系统；后续提文案、拆结构、判断适合账号和生成同类型内容由 Codex/AI 处理。

## 核心模块

- 今日中控台：按对接咨询/负责人汇总今日任务，明确每条内容发给哪个兼职、由谁对接；同时显示今日链路复盘，可一键复制工作简报，并可查看待生成、待选择、素材阻塞、待交付、待核对、待爆款分析、图片任务、剪辑任务和异常账号。
- AI 工作包：把当前首页状态、Codex 下一步优先级、待爆款分析、图片/剪辑任务、异常账号和本地路径导出为 Markdown，方便交给 Codex/Claude 继续判断。
- 运营建议：异常账号会给出下一步动作，例如补排期、处理素材缺口、加爆款提权或优先核对。
- 排期规划：按未来 7/14/30/60 天查看所有账号内容安排，并直接处理生成、交付、派发和核对动作。
- 案例库：新建、批量导入、搜索筛选、删除数据库记录。
- 爆款链接：工作人员只贴链接；完成青豆提取和结构分析后再批量生成账号人设化版本。
- 本地交付包：生成可直接通过电脑微信发送的文案、图片、视频任务文件夹；如果没有可用素材，任务会进入 `素材阻塞`，不会误进入可交付。
- 本地交付包会包含素材顺序清单，方便按顺序拖进微信发送。
- 视频/口播交付包会额外生成口播字幕、剪辑要求和成片回收说明。
- 图片任务：预留 Image v2 接入，`IMAGE_API_KEY` 为空时进入 `waiting_key`，同时生成本地图片任务说明和可复制 Prompt。
- 剪辑任务：生成剪辑任务单，可在首页复制任务单、打开目录并标记状态。
- 核对队列：打开抖音链接后人工/半自动回填播放、点赞、评论、粉丝数据，并可复制核对清单。

## 环境变量

在 `app/.env` 里预留：


当前 key 为空时，文案、排期、交付包和核对流程仍可正常使用。
