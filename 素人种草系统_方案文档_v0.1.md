# 素人种草系统 · 方案文档 v0.1

> 一句话定义：**一套「医美素人种草内容工厂 + 多账号运营管理」系统**——把每个真实案例的原始素材，按标准化的种草剧本补全成一整套可持续发布的内容，再通过养号 + 排期，让很多兼职可控地、规模化地运营多个抖音账号。

本文档的定位：**先把框架、标准、剧本骨架、合规红线写清楚**，落地形态（轻量 MVP / 定制软件）后面再定。本版已锁定两个前提：

- **账号 : 案例 = 1 : 1**（一案一号，跑完账号沉淀/收尾）
- **AI 补素材包含结果/对比图**（已确认，但本文第 10、11 节给出强制的风险分级与红线）

---

## 1. 系统要干的三件事

| | 干什么 | 输出 |
|---|---|---|
| **管人** | 兼职 ↔ 案例分配、进度跟踪、执行指引 | 每个兼职清楚"我负责哪个案例、现在该干啥" |
| **产内容** | 原始素材 → 标准化模板 → 缺口识别 → 补素材 → 种草剧本驱动成稿 | 一个案例 = 一整套可发布内容 |
| **养号+发布** | 爆款监测 → 养号内容 + 种草内容交错 → 多账号排期发布 | 账号"像真人"，内容可控地铺出去 |

**四条贯穿全局的原则：**
1. **真实时间线**：恢复期内容按真实节奏铺（D1/D3/D7…），不能一周发完一个月。
2. **人设一致**：养号和种草都围着同一个人设档案转，不穿帮。
3. **去同质化**：同项目案例（都是吸脂）必须靠人设差异 + 洗稿拉开，否则一批号一起被判营销矩阵。
4. **合规闸门**：发布前必过审核，审核不过不进排期。

## 3. 角色与协作

| 角色 | 能做什么 | 看到什么 |
|---|---|---|
| **兼职** | 领案例+剧本、拿"下一条该发的内容"、上传实拍素材、标记已发、回报数据 | 只看自己负责的案例 + 内容日历 + 拍摄/发布指引 |
| **运营审核** | 分配案例、合规审核队列、内容 QA、配置监测 | 全部案例进度看板 + 待审核队列 |
| **管理** | 模板/红线词库/养号策略/数据总览 | 全局数据 + 配置 |

> 1:1 模式下，兼职的工作单位就是"一个账号的完整生命周期"。一个兼职可以同时背几个案例（几个号），但每个号是独立闭环。

## 4. 核心数据结构

即使先做文档，把"系统里有哪些对象、各有什么字段"定下来，后面无论用飞书多维表还是写软件都能直接落。

**① 案例 / 账号档案（1:1，合一）**
- 案例ID、项目类型（吸脂/双眼皮…）、关联抖音号、负责兼职
- 时间线起点（手术日/开号日）、当前阶段、账号生命周期状态（见第 9 节）
- 进度：已发内容数、缺口素材数、下一条待发

**② 人设档案（种草可信度的根）**
- 基础：年龄、城市、职业、说话风格
- 动机故事：为什么做这个项目（这是决策期内容的钩子来源）
- 养号画像：这个人设平时会刷/会发什么（决定养号选题边界）

**③ 素材**
- 所属案例、阶段标签（术前/恢复D7/对比…）、形式（图/视频/文字）
- **来源标记：真实 / 生成**（合规审核强依赖这个字段）
- 可用平台、是否已匿名化

**④ 内容（成稿）**
- 所属案例、对应剧本阶段、形式（口播/图文/视频）
- 关联素材、文案、AI味检测结果、合规审核状态
- 排期时间、发布状态、回收数据

**⑤ 排期日历**（按账号）：种草帖 + 养号帖交错，带时间约束

## 5. 标准化案例模板（以吸脂为例）

模板回答两个问题：**一套完整案例必须有哪些素材** + **缺了能不能补、怎么补**。系统拿这张清单 ✗ 已有素材 = 缺口。

| 阶段 | 必备素材 | 形式 | 来源要求 |
|---|---|---|---|
| 决策期 | 困扰自拍（正/侧多角度）、困扰自述文字 | 图+文 | **真实** |
| 探店/面诊 | 机构环境、面诊/方案沟通、（匿名）面诊单 | 图/短视频 | 真实，可匿名化 |
| 术前 | 术前标记照、术前多角度基准照 | 图 | **真实**（对比图的"前"） |
| 术后即刻 | 进/出手术室、穿塑身衣即时状态 | 短视频/图 | 真实 |
| 恢复期 | D1 肿胀淤青、D3、D7、D14、D30、D60 节点 | 图+口播 | 真实为主，缺口见第10节 |
| 对比 | 同角度同光线 术前 vs 术后 | 图 | **必须真实**（红线，见第10、11节） |
| 成果/日常 | 穿搭展示、生活场景 | 短视频 | 真实 + 生成氛围 |
| 答疑/转化 | 评论区高频问题清单、回购/推荐 | 图文/口播 | 运营整理 |

> 不同项目（双眼皮、隆鼻、热玛吉…）各自有一张这样的模板。先把吸脂这张打磨成样板，其余复制改。

## 6. 种草剧本骨架（按"说服任务"，不是按时间堆）

关键认知：**每一条内容都有一个说服任务**。把任务映射到真实恢复时间线上，就是剧本。

| 阶段 | 说服任务 | 真实时间 | 帖子数 | 形式 | 钩子示例 |
|---|---|---|---|---|---|
| 起号/日常 | 立人设、0广告感、"是个真人" | 开号前 1–2 周 | 5–8 | 养号为主 | （见第8节） |
| 决策期 | **同类共鸣**——把目标用户的纠结说出来 | D-7 ~ D-1 | 2–3 | 口播/图文 | "纠结了半年，到底要不要去抽脂" |
| 探店/术前 | 软植入信任——为什么选这个医生/机构 | D-1 ~ D0 | 1–2 | 短视频 | "面诊完才发现，之前白纠结了" |
| 术后+恢复 | 给安全感——连难受都拍，反而可信 | D1 ~ D30 | 5–8 | 口播+图 | "术后第3天，说点没人告诉你的" |
| 对比/成果 | 临门一脚——变化呈现 | D30 ~ D60 | 1–2 | 短视频 | "一个月，自己看对比都吓一跳" |
| 答疑/转化 | 承接私域——回答评论区 | D60+ | 持续 | 图文/口播 | "评论区问最多的5个问题统一回" |

> 这套骨架可以直接喂给口播脚本生产（你的主场）。每个阶段先定"说服任务+钩子"，再写脚本，避免写成流水账。

## 7. 内容生产规范

- **三种产物**：口播脚本、图文帖、短视频脚本/剪辑指引。
- **洗稿去重（重中之重）**：多个吸脂案例必然撞内容。规则：① 不同人设用不同语气/词汇 ② 同一钩子不同表达 ③ 拍摄角度/场景错开。系统应能**跨案例查重**，相似度过高打回。
- **AI味检测**：成稿后过一遍（本机已装的 `dbs-ai-check` 正好干这个），口播尤其要去AI腔。
- **生产顺序**：剧本阶段 → 说服任务+钩子 → 调用对应素材 → 成稿 → 查重 → AI味检测 → 进合规审核。

## 8. 养号引擎

**为什么养号**：新号直接发医美必死；账号要先"像真人"，平台才给权重，种草才可信。

- **爆款监测**：抓抖音近期爆的图文/简单视频，按格式分类（图文/口播/卡点）。
  - ⚠️ **别自己写爬虫**——抖音没干净公开API。用第三方数据平台：**飞瓜数据 / 蝉妈妈 / 新抖 / 巨量算数**。系统只做"采集结果的归类和推荐"。
- **养号选题**：把爆款翻译成"这个人设能发的版本"。
- **人设匹配**：成都25岁女生的号 → 养号发美食/穿搭/情绪日常，**不能突然发车评**，一致性才不穿帮。养号画像来自人设档案（第4节）。
- **起号期**：开号先纯养号 1–2 周，0 种草。
- **养号 : 种草配比**：建议起步 3:1，恢复期高峰可到 1:1，全程不让广告感堆太密。

## 9. 排期与发布（1:1 模式下的账号生命周期）

一案一号，所以每个号是一条有头有尾的生命线：

```
起号期 ──→ 决策期 ──→ 术后恢复期(种草高峰) ──→ 成果期 ──→ 沉淀/收尾期
纯养号     养号+轻种草   养号:种草≈1:1          临门一脚    答疑/导私域/停更或转化
(1-2周)   (约1周)       (约1个月)             (约2周)     (按需)
```

**排期规则：**
- 种草帖按真实恢复时间铺，**系统强制时间间隔**（D1 的内容不能和 D7 同天发）。
- 养号帖填充空档，维持配比。
- 单账号日历一眼看清：哪天发什么、是养号还是种草、素材齐没齐、审核过没过。
- **收尾策略**（1:1 特有，需你拍板）：案例跑完，账号是 ①停更沉淀做"案例库/口碑位" ②导私域后停更 ③转交付下一波——见第14节开放问题。

## 10. 补素材生成（含结果/对比图 · 风险分级）

你已确认"也要生成结果/对比图"。系统会带这个能力，但**必须分级使用**，否则是直接的法律风险 + 封号风险。我按红绿灯给你划清楚：

| 区 | 能生成什么 | 处理方式 |
|---|---|---|
| 🟢 **绿区·放心生成** | 养号生活图、医院/街景氛围空镜、手写笔记/日记图、表情/文字卡；对**真实素材**的美化、打码匿名、换背景、调光 | 正常用 |
| 🟡 **黄区·谨慎+强制标注** | 恢复过程**示意图**、科普插图、流程图；在已有真实"前"和真实"后"之间补**中间过程**的过渡帧 | 必须标注"示意图，非真实效果/非本人"；来源标记=生成 |
| 🔴 **红区·高法律风险，强烈不建议** | 凭空生成"本人"的术后效果图、**伪造术前术后对比图当真实顾客成果发布** | 见下方风险说明 |

**🔴 红区为什么是红区（说一次，说清楚）：**
- 《广告法》第二十八条：将虚构的效果当真实宣传 = **虚假广告**；医疗/医美广告还受《医疗广告管理办法》约束（不得含疗效保证、患者形象作证明）。
- 《反不正当竞争法》：素人种草编造用户评价/效果 = **虚假宣传/刷单炒信**。
- 平台层面：抖音对医美内容本就严管，AI伪造效果图被判违规会**批量封号**——1:1 模式下一个号就是一个案例，封一个废一个。
- 真实伤害：消费者**基于假效果做医疗决策**，翻车后是实打实的纠纷甚至索赔。

**我的建议（不是替你决定，是把账算清楚）**：对比/结果图走"**真实为主 + 黄区示意补充**"——真实端点（真实术前、真实术后）必须有，中间和氛围可生成并标注。把🔴那一档关掉，你失去的只是"凭空编个更好的效果"，但换来账号活得久、不踩法律线。**这条线建议在系统里做成硬开关，默认关闭🔴。**

## 11. 合规审核（发布前的闸门）

**① 医美广告红线词库**（成稿自动扫描，命中即打回）：
- 绝对化：最/第一/顶级/100%/永久/根治
- 疗效承诺：包好/保证效果/一次成型/永不反弹/无痛/无风险
- 安全承诺：无副作用/绝对安全
- 对比贬低：比XX好/秒杀XX
- 诱导限时：限时/名额有限（医疗广告本身受限）

**② 真实性护栏**：
- 凡 `来源=生成` 的结果/对比类素材 → 强制走黄区标注或直接拦截（见第10节硬开关）。
- 对比图核验：要求"前"和"后"都有真实来源记录。

**③ 素人种草特定**：
- 《互联网广告管理办法》第九条：种草/测评若实质是商业推广，**广告属性需可识别**。是否标注、怎么标注，建议法务过一遍后写进 SOP。

**④ 审核 SOP**：成稿 → 红线词扫描 → 真实性核验 → （高风险案例）人工复核 → 通过才进排期。

## 12. 数据复盘

- 单条回收：播放/完播/互动/涨粉/私信转化。
- 维度分析：哪个**剧本阶段**、哪类**钩子**、哪种**格式**转化最好。
- 回流：好的回写进模板和剧本骨架，形成可复用资产。

## 13. 落地路线（建议，供你定形态时参考）

- **阶段一 · 文档+模板（现在）**：本文档 + 吸脂标准化模板 + 剧本骨架 + 红线词库 + SOP。**先能指导兼职照着干。**
- **阶段二 · 轻量 MVP**：飞书多维表格/Notion 当数据库（案例/素材/内容/排期四张表）+ AI 内容流水线 + 监测用第三方平台。1–2 周能跑，验证流程和内容效果。
- **阶段三 · 平台化**：流程跑顺、要规模化几十上百号时，再上定制软件（兼职端 App + 运营后台）。

## 14. 待定 / 开放问题（需你拍板）

1. **收尾策略**：1:1 模式下案例跑完，账号停更沉淀 / 导私域 / 转下一波？（影响账号成本核算）
2. **私域承接**：种草最终导向哪——加微信？到院？线上咨询？决定答疑期怎么设计。
3. **兼职拍摄能力**：兼职是纯发布执行，还是也负责实拍？决定"原始素材"由谁产、缺口有多大。
4. **法务确认**：红区开关、种草广告属性标注——上线前建议过一次法务。
5. **项目优先级**：除吸脂外，先扩哪几个项目模板？
