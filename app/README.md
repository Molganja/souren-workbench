# 素人种草运营工作台 v1

本地网页工作台，用于管理兼职账号、三类内容排期、候选稿、素材扫描、网页内微信交付、图片任务、账号数据监控和爆款提醒。

## 启动

最简单方式：双击项目根目录的：

```text
启动素人工作台.command
```

它会自动检查环境、安装依赖、构建前端、启动本地服务并打开浏览器。

命令行开发方式：

```bash
cd /Users/licc/Desktop/素人系统/app
npm install
npm run dev
```

- 前端：`http://127.0.0.1:5173`
- 后端/API：`http://127.0.0.1:5174`

桌面窗口方式：

```bash
cd /Users/licc/Desktop/素人系统/app
npm run client
```

生成 Mac `.app` 目录：

```bash
cd /Users/licc/Desktop/素人系统/app
npm run client:mac
```

打包输出在 `app/release/`。客户端包不会包含真实 `.env`、数据库和素材。Mac `.app` 会用 Electron 内置 Node 启动本地后端并打开工作台窗口，不需要工作人员手动启动服务。后续如果要做 Windows 分发，需要在 Windows 机器上跑对应打包验收。

生产构建后也可以只跑后端：

```bash
npm run build
npm run start
```

打开：`http://127.0.0.1:5174`

## 本地数据

系统会自动创建：

```text
/Users/licc/Desktop/素人系统/data/souren.sqlite
/Users/licc/Desktop/素人系统/素材库/真实案例/
```

新建案例后，每个案例会自动生成：

```text
00-原始素材/
01-已筛选素材/
02-生成补充/
03-交付给兼职/
04-发布回收/

```

网页端只保存结构化数据和本地路径，大图片、视频和交付文件都放在本地素材库；日常发送动作在网页内完成。

## 图片接口

`app/.env` 里预留：

```env
PORT=5174
SOUREN_HOST=127.0.0.1
SOUREN_ACCESS_CODE=
IMAGE_API_KEY=
IMAGE_API_URL=
IMAGE_MODEL=image-v2
IMAGE_SIZE=1024x1024
DOUYIN_COLLECTOR_URL=
```

当前 v1 会创建图片任务和提示词。`IMAGE_API_KEY` 或 `IMAGE_API_URL` 为空时，任务显示为“待接入图片接口”，不会影响其他流程；两者都填好后可在网页内生成图片、预览并下载。

如果工作人员在另一台电脑上使用，把主机 `app/.env` 改成：

```env
SOUREN_HOST=0.0.0.0
SOUREN_ACCESS_CODE=给工作人员的访问码
```

重启后到「系统配置」复制局域网网址。工作人员端只打开网页、输入访问码；密钥、数据库、素材库仍在主机本地。远端工作人员不会看到打开主机本地目录的按钮，日常对接只用复制、下载和标记。

## 当前闭环

1. 工作人员打开「今日中控台」，只看今天该处理的账号和动作。
2. 新建案例并生成本地案例目录。
   - 新建时会随机生成人设，并自动生成近期排期。
   - 工作人员默认只需要填兼职微信昵称、抖音主页/作品链接和项目；人设、抖音号、对接人都可以后续编辑。
   - 也可以在案例库使用「批量导入」，一次录入多名兼职/账号，并自动生成排期。
3. 把原始素材放进 `00-原始素材/`，点击扫描素材。
4. 系统生成排期槽位，对当天或近期槽位生成 3 条候选稿。
5. 工作人员选择一条并锁定。
6. 生成微信交付内容。
   - 如果没有可用素材，任务会进入「素材阻塞」，不会误进入可交付。
   - 素材阻塞任务可以直接复制补素材说明，补齐后重新生成交付内容；本机管理端才显示打开素材目录入口。
   - 如果是视频内容，可以从排期或案例详情创建「剪辑任务」，系统会生成本地剪辑任务单。
7. 工作人员点「查看交付内容」，在网页内复制文案、预览/下载图片或视频，用电脑微信发给兼职。
8. 标记派发；确认兼职或剪辑人员完成发布/交付后标记完成。
9. 账号数据由系统按 24 小时周期生成 Chrome 采集清单；我用已登录抖音的 Chrome 逐个查看主页，再在网页里点「回填数据」后，系统自动统计初始粉丝、最新粉丝和作品播放/点赞/评论。
10. 作品达到爆款阈值后，首页提醒安排助理号/阑尾号去评论区互动。
11. 工作人员在「爆款链接」里只粘贴抖音爆款作品链接；内容提取、拆结构和生成同类型内容由系统使用者处理。

日常操作手册见项目根目录：`日常运营SOP.md`。

## 验收

```bash
cd /Users/licc/Desktop/素人系统/app
npm run doctor
npm run check
npm run client:check
npm run test:e2e
npm run verify
npm run verify:github
```

`test:e2e` 会临时创建测试数据库和素材目录，跑完后自动清理。
`verify` 会顺序运行 doctor、build、E2E，并额外启动隔离服务检查 `/api/health` 和 `/api/readiness`。
`verify:github` 会把远端主分支是否与本地提交同步作为硬性验收；如果本机没有写权限，它会失败并提示需要认证后上传。

## 新增实用操作

- 「排期规划」页可以按未来 7/14/30/60 天、状态、内容类型和账号关键词查看全局排期，并直接处理生成、交付、派发和完成确认。
- 「今日中控台」会按对接咨询/负责人汇总今日任务，工作人员能直接知道每条内容该发给哪个兼职、由谁对接。
- 「今日链路复盘」按状态串起待生成、待选择、已锁定、素材阻塞、可交付、已派发、已完成，并支持复制今日工作简报。
- 「系统配置」页只显示工作人员能理解和使用的状态：局域网访问、素材目录、图片接口、内容阶段比例和素材模板。
- 「今日中控台」会直接显示素材阻塞、待爆款分析、图片任务、剪辑任务、账号采集状态和账号数据动作。
- 「今日中控台」支持一键准备今日交付：生成候选、锁定稳妥稿、生成交付内容；缺素材的任务会进入素材阻塞，不会计入可交付。
- 案例库支持批量导入兼职/账号，适合一次录入 40+ 账号。
- 案例库支持按关键词和健康状态搜索筛选，也可以删除误建案例；删除时会同步删除本地案例素材目录。
- 案例详情里本机管理端可以直接「打开素材目录」；局域网工作人员只在网页内复制说明、查看交付内容和标记状态。
- 案例详情里可以编辑案例信息，也可以手动新增临时内容槽位。
- 案例详情会按项目模板显示素材缺口和账号健康提示。
- 异常账号会显示运营建议，把断更、素材缺口、账号数据超 24 小时未更新、播放下滑和爆款提醒转成下一步动作。
- 「账号数据动作」会按优先级汇总待采集、爆款互动、偏冷补内容和增长承接，工作人员按这个队列处理账号表现。
- 交付内容生成后，可以在排期卡片里直接「查看交付内容」，在网页内复制文案、预览/下载图片或视频。
- 候选稿卡片可直接复制执行说明和抖音发布文案，方便发微信。
- 排期卡片可以标记异常，避免卡在无法推进的内容上。
- 素材扫描后，可以把素材标记为「可用 / 需处理 / 不可用」。
- 素材缺口旁可以直接创建对应图片任务，也可以复制补素材说明发给素材整理人员。
- 账号数据监控会记录初始粉丝、最新粉丝、作品播放/点赞/评论；默认使用 Chrome 登录态采集清单和网页「回填数据」表单，未采集时只显示待采集或等待 Chrome 采集，不会伪造数据。
- 「爆款链接」页只要求工作人员粘贴抖音作品链接；系统先记录为待分析，首页也会显示待分析链接，后续由系统使用者提取内容、拆结构，再按适合/禁用人设关键词批量生成。
- 待爆款分析链接可以复制分析任务，任务文本包含链接、提取字段、回填标准和下一步。
- 图片任务可以复制提示词；接入图片接口后可以直接在网页内生成、预览、下载，并标记“待审核 / 已通过 / 已驳回”。
- 剪辑任务会生成 `剪辑任务单.txt`，可复制给剪辑人员；剪辑/发布完成后在首页把任务标记为完成即可。
- 交付内容会生成 `05-素材顺序清单.txt`，按顺序说明微信要发送的图片或视频。
- 视频/口播交付内容会额外生成 `03-口播字幕文案.txt`、`04-剪辑要求.txt` 和 `07-成片回收说明.txt`。
- 图片任务仍会在本地保存 `图片任务说明.txt` 和生成图片，但工作人员主要在网页内查看和下载。
- 爆款互动动作会展示作品链接、播放/点赞/评论和互动建议，工作人员只需安排助理号/阑尾号互动并标记已处理。
