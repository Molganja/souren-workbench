# 素人种草运营工作台 v1

本地网页工作台，用于管理兼职账号、三类内容排期、候选稿、素材扫描、网页内微信交付、图片任务、账号数据监控和爆款提醒。

## 启动

最简单方式：双击项目根目录的：

```text
启动素人工作台.command
```

它会自动检查环境、安装依赖、构建前端、启动本地服务并打开浏览器。

命令行开发方式：

```bash
cd /Users/licc/Desktop/素人系统/app
npm install
npm run dev
```

- 前端：`http://127.0.0.1:5173`
- 后端/API：`http://127.0.0.1:5174`

桌面窗口方式：

```bash
cd /Users/licc/Desktop/素人系统/app
npm run client
```

生成 Mac `.app` 目录：

```bash
cd /Users/licc/Desktop/素人系统/app
npm run client:verify
```

打包输出在 `app/release/`。客户端包不会包含真实 `.env`、数据库和素材。Mac `.app` 会用 Electron 内置 Node 启动本地后端并打开工作台窗口，不需要工作人员手动启动服务。`client:verify` 会先生成 `.app`，再实际启动打包后的客户端，确认内置后端和独立工作数据目录可用。后续如果要做 Windows 分发，需要在 Windows 机器上跑对应打包验收。

生产构建后也可以只跑后端：

```bash
npm run build
npm run start
```

打开：`http://127.0.0.1:5174`

## 本地数据

系统会自动创建：

```text
/Users/licc/Desktop/素人系统/data/souren.sqlite
/Users/licc/Desktop/素人系统/素材库/真实案例/
/Users/licc/Desktop/素人系统/素材库/服务器案例库/
/Users/licc/Desktop/素人系统/素材库/通用素材/
```

新建案例后，每个案例会自动生成：

```text
00-原始素材/
01-已筛选素材/
02-生成补充/
03-交付给兼职/
case.json
```

网页端只保存结构化数据和本地路径，大图片、视频和交付文件都放在本地素材库；日常发送动作在网页内完成。

共享盘/服务器目录是工作人员放案例原始素材的入口。可以在案例登记时直接填写「共享原始素材路径」，也可以把 `SOUREN_CASE_LIBRARY_ROOT` 指到服务器上的“待分配案例根目录”，然后在案例库页点击「扫描服务器案例库」一键登记。登记时系统会校验该目录、自动同步一次到本地案例目录，并记录来源路径；后续共享目录新增文件时再补同步。

医院素材、套图素材和备用素材这类跨案例可复用素材放进 `素材库/通用素材/` 或 `SOUREN_SHARED_MATERIAL_ROOT` 指向的目录，在运行状态页扫描。案例素材和通用素材分开管理，后续图片生成可以同时引用“案例人物/过程素材”和“医院/通用场景素材”。

素材扫描会自动按目录名和文件名分类：案例人物、案例过程、案例对比、案例场景、医院素材、套图素材、生活场景、文字卡等。图片任务会把案例人物/变化参考和医院/通用场景素材一起写进参考素材，后续接 Image2 时可用于人物与场景合成。

## 图片接口

`app/.env` 里预留：

```env
PORT=5174
SOUREN_HOST=127.0.0.1
SOUREN_ACCESS_CODE=
SOUREN_CASE_LIBRARY_ROOT=
SOUREN_SHARED_MATERIAL_ROOT=
DOUYIN_COLLECTION_CHECK_INTERVAL_MS=21600000
DOUYIN_COLLECTOR_AUTO_RUN=1
DOUYIN_COLLECTOR_LIMIT=10
DOUYIN_COLLECTOR_WAIT_MS=6000
DOUYIN_COLLECTOR_COOLDOWN_MS=900000
IMAGE_API_KEY=
IMAGE_API_URL=
IMAGE_MODEL=gpt-image-1
IMAGE_SIZE=1024x1024
```

当前 v1 会创建图片任务和提示词。`IMAGE_API_KEY` 为空时，任务显示为“待接入图片接口”，不会影响其他流程；只填 `IMAGE_API_KEY` 就会使用默认图片生成端点，`IMAGE_API_URL` 只在接其他供应商或本地服务时填写。

如果工作人员在另一台电脑上使用，主机进入「运行状态」，点击「开启局域网访问」。系统会生成访问码并写入主机配置，重启后到「运行状态」查看局域网网址，再把网址和访问码发给工作人员。

工作人员端只打开网页、输入访问码；密钥、数据库、素材库仍在主机本地。用完后可在「运行状态」点击「关闭局域网访问」，重启后工作人员端网址会失效。系统不提供本地目录直达入口，工作人员只用查看任务、复制、下载和标记。系统不会允许没有访问码的局域网开放。

## 当前闭环

1. 工作人员打开「今日中控台」，只看今天该处理的账号和动作。
2. 新建案例并生成本地案例目录。
   - 新建时只填兼职微信昵称、抖音主页/作品链接、项目和共享原始素材路径。
   - 微信昵称、抖音主页/作品链接和共享原始素材路径必须填写，系统从源头避免创建无法采集数据、无法同步素材的空账号。
   - 系统自动生成近期排期；内部写作参考由后台处理，工作人员不用在新建或编辑时填写额外资料。
   - 也可以在案例库使用「批量导入」，一次录入多名兼职/账号，并自动生成排期。
3. 工作人员把挑好的素材放进共享路径；如果是服务器待分配案例库里的目录，可在案例库先「扫描服务器案例库」并登记。登记时系统会自动同步一次素材。
4. 后续共享路径又放入新素材时，再点「同步共享素材」，然后扫描素材。
5. 系统生成排期槽位，对当天或近期槽位生成 3 条候选稿。
6. 工作人员选择一条并锁定。
7. 生成微信交付内容。
   - 如果没有可用素材，任务会进入「素材阻塞」，不会误进入可交付。
   - 素材阻塞任务回到案例详情查看素材缺口，补齐后重新生成交付内容；素材路径只作为后台保存位置，不作为工作人员日常入口。
   - 图片任务只从素材缺口或明确排期用途创建，不提供泛化新建入口。
   - 如果是视频内容，可以从排期或案例详情创建「剪辑任务」，剪辑要求直接在网页内展示。
8. 工作人员点「查看交付内容」，弹窗会锁定当前收件微信；先点「确认只发给这个微信」，再按顺序复制文案、下载图片或视频，用电脑微信发给兼职。第一次给这个兼职发内容时才显示可复制的「兼职须知」，后续交付只保留已发过提示；发布要求已写进「发给兼职文案」，不再单独复制。标记已派发后，交付文案和素材下载都只读，不再显示复制或下载按钮。
9. 发完微信后，在交付弹窗里点「已用微信发送」；确认兼职或剪辑人员完成发布/交付后标记完成。
10. 账号数据由系统按活跃度登记采集任务；正在投放或起量的账号勤采，偏冷账号降频，休眠账号一周一次。到期账号会按待首采、起量、活跃、偏冷、休眠排序，不按账号数硬跑。主机后台会自动用已登录抖音的 Chrome 查看到期主页并把数据写入后台；解析不到真实数据时会跳过，不写空结果。
11. 作品达到爆款阈值后，首页提醒安排助理号/阑尾号去评论区互动。
12. 工作人员在「爆款链接」里只粘贴抖音爆款作品链接；链接进入待分析状态，主机侧写回内容提取和结构拆解后再生成同类型内容。

系统会用账号采集数据反向调整排期：连续低播放账号自动降为两天一条或暂停自动补新排期；已生成候选、已锁定或可交付但还没派发的旧任务会同步撤下，避免工作人员每天被平账号任务压着走。

如果账号已派发超过 3 天还没有确认完成，系统会按「失联暂停」自动处理：普通待发、补素材、剪辑和采集队列都会撤下，不再为这个账号生成新的首页待办。

如果兼职后来恢复配合，在案例详情点击「恢复账号」即可。系统会把旧的超时已派发任务标为「已取消」，再补未来 7 天的基础排期，避免工作人员手动改状态和补排期。

后台默认每 6 小时检查一次是否有账号到期；这只是检查间隔，不代表所有账号都会进入采集清单。已经等待 Chrome 的账号不会重复排队。

主机侧采集执行器会由后端定时自动启动；下面命令只用于排查：

```bash
cd /Users/licc/Desktop/素人系统/app
npm run collect:douyin -- --dry-run --limit 3
npm run collect:douyin -- --register --limit 10
```

第一次自动运行前确保 Google Chrome 已登录抖音，并允许 Apple Events 执行 JavaScript。`--dry-run` 只看解析结果不写回；`--register` 会先把到期账号登记为等待 Chrome 采集，再把解析到的真实账号/作品数据写回后台。

演示账号或 `demo_` 这类测试抖音链接只用于界面验收，不会进入自动采集队列。

日常操作手册见项目根目录：`日常运营SOP.md`。

## 验收

```bash
cd /Users/licc/Desktop/素人系统/app
npm run doctor
npm run collect:douyin:self-test
npm run check
npm run client:check
npm run test:e2e
npm run verify
npm run verify:github
```

`test:e2e` 会临时创建测试数据库和素材目录，跑完后自动清理。
`verify` 会顺序运行 doctor、build、E2E，并额外启动隔离服务检查 `/api/health` 和 `/api/readiness`。
`verify:github` 会把远端主分支是否与本地提交同步作为硬性验收；失败时按终端提示处理认证、提交或推送。

## 新增实用操作

- 「排期规划」页只读，可以按未来 7/14/30/60 天、状态、内容类型和账号关键词查看全局排期；生成、交付、派发和完成动作只能回到「今日操作队列」队首后处理。
- 「数据复盘」只看趋势、账号表现和采集结果，不提供打开案例的执行入口；具体处理仍回到「今日操作队列」。
- 「今日中控台」只保留一张「今日操作队列」，把微信交付、生成交付、选候选、生成候选、素材同步、爆款互动和剪辑任务按优先级放在同一个入口里；当前只完整显示队首账号，后面的任务只显示锁住数量。
- 「后续账号预览」默认折叠，按今日操作队列聚合到兼职/账号名下，不只看排期槽位；素材同步、剪辑、爆款互动和账号策略动作都会显示到对应兼职名下，但不重复展开同一条任务的操作按钮。
- 「异常账号」默认折叠成提醒，不在首页直接铺开；真正处理仍等异常动作排到今日队首。
- 历史遗留账号如果缺抖音主页或共享原始素材路径，会进入「补登记」队列；编辑案例也只改微信、抖音、项目和共享素材路径，明确提交共享素材路径时会校验目录是否存在。
- 新建、批量导入、服务器案例库登记或补登记编辑完成后，账号会自动排入 Chrome 采集队列；工作人员不需要去运行状态里手动登记采集。
- 「运行状态」页默认只显示能理解和使用的状态：验收清单、局域网访问、通用素材扫描和图片生成状态；通用素材校准、素材保存位置、账号采集、爆款分析和图片生成排队统一放进折叠区，账号采集由后台自动排队，不要求工作人员点击。
- 「今日中控台」不把账号采集这类后台动作放到首屏；采集状态只放在「运行状态」里的折叠「系统后台」。
- 「今日中控台」会显示待同步共享素材；共享目录有新文件但还没同步到本地案例目录时，可直接在首页一键同步。
- 「今日中控台」不提供一键批量准备入口；生成候选、锁定文案和生成交付都只能围绕当前队首任务推进，避免一次制造多条待发送内容。
- 案例库支持批量导入兼职/账号，适合一次录入 40+ 账号；导入会整批预检共享目录，通过后自动同步素材和生成排期，避免半导入。
- 案例库支持扫描服务器案例库，把共享盘/服务器里的待分配案例目录登记成案例；登记时用网页弹窗绑定微信、抖音、项目和共享素材路径，同时同步素材并生成近期排期。
- 案例库支持按关键词和健康状态搜索筛选，也可以删除误建案例；删除时必须输入“删除 + 微信昵称”确认，确认后会同步删除本地素材副本。
- 案例详情只在网页内复制说明、预览和下载素材、查看交付内容和标记状态；本地素材目录只作为后台保存和读取位置。
- 案例详情支持「共享原始素材路径」和「同步共享素材」；共享盘/服务器只作为素材入口，系统生成交付使用本地受控副本。
- 运行状态页支持扫描通用素材库，医院素材、套图素材和备用素材可以跨案例复用；扫描后只在折叠的「通用素材校准」里预览并修正分类、用途和可用状态，不提供发送下载入口。
- 素材清单可以手动标记用途，例如案例人物、案例过程、案例对比、案例场景和日常素材，修正自动分类偏差。
- 案例详情里可以编辑案例信息；临时内容槽位由首页策略动作或后台维护生成。
- 案例详情会按项目模板显示素材缺口和账号健康提示。
- 异常账号会显示运营建议，把断更、素材缺口、播放下滑和爆款提醒转成下一步动作；未采集状态只放在「运行状态」的折叠「系统后台」。
- 「账号策略动作」会按优先级汇总爆款互动、偏冷补内容和增长承接；偏冷和增长动作只会按账号投放频率找下一个空档生成，不给休眠、暂停或当天已有排期的账号额外加任务。
- 交付内容生成后，只有当前队首任务可以打开交付内容并复制/派发；不是队首的排期卡片只显示“排到今日队列队首后处理”。列表页不提供直接派发或完成按钮，必须进交付弹窗按固定步骤发送并确认完成；没有可发送图片或视频时不能标记已派发；已派发内容只读，避免重发。
- 任务进入交付链路后，候选稿和交付包都会锁住；如果已经可交付、已派发或已完成，系统不会允许重新生成交付或改选候选。
- 候选稿卡片只负责选择和锁定内容；文案复制集中在「查看交付内容」弹窗里，避免重复入口。
- 同一排期只允许生成一个剪辑任务；已建任务会在排期卡片显示「剪辑任务已建」，避免重复派给剪辑。
- 首页「今日操作队列」只完整显示第一条任务，后面的任务只汇总数量和类型；任务标记完成后从今日队列消失，队列清空后显示「今天发完了」。已发出但等回复的任务放进折叠的「等待兼职确认」，收到回复后再展开收尾。
- 首页「后续账号预览」和「异常账号」只做概览；非队首账号不显示打开按钮，只提示排到队首后打开。
- 后台准备、生成候选和生成交付接口也只处理当前队首；当前面还有待发送、爆款互动、素材同步、失联确认或异常任务时，不会越过它去批量生成后面的交付内容。
- 案例详情不再绕过今日队列执行任务；排期规划只读。非队首任务只能查看，不能复制、派发、完成、生成候选、选择候选、生成交付、素材同步/扫描/标记、创建或检查图片任务、完成剪辑或标记爆款互动。后端普通接口调用也会拒绝绕过队首推进任务。
- 交付前的排期卡片可以标记异常，避免卡在无法推进的内容上；可交付、已派发和已完成任务不能再打回异常。
- 素材扫描后，可以把素材标记为「可用 / 需处理 / 不可用」。
- 素材缺口旁可以直接创建对应图片任务；图片任务必须带明确用途，补素材动作集中在案例详情里处理。
- 账号数据监控会记录初始粉丝、最新粉丝、作品播放/点赞/评论；默认使用 Chrome 登录态采集任务，主机端采集完成后写入后台。未采集状态不进入首页首屏和异常账号，不会伪造数据。
- 「系统后台」只放在运行状态并默认折叠，聚合待分析爆款、到期 Chrome 采集和图片任务；到期账号自动排队，不进入工作人员今日操作队列。
- 「爆款链接」页只要求工作人员粘贴抖音作品链接，链接必填；系统先记录为待分析，待分析链接不进入今日操作队列。主机侧通过 `/api/viral-templates/:id/analysis-result` 写回内容提取和结构拆解后，再按账号匹配规则给可投放账号批量生成。
- 待爆款分析链接只保留原链接和待分析状态，避免把分析流程变成工作人员要处理的前台待办；暂停、休眠、不符合投放频率或已有同日任务的账号会自动跳过。
- 图片任务在网页内展示提示词；接入图片生成后可以直接生成、预览、下载，并标记“待检查 / 可用 / 不用”。图片任务只来自素材缺口或具体排期，不让工作人员临时建空任务。
- 剪辑任务会在网页内展示剪辑填空卡和固定剪辑配方；排到首页队首后先打开完整剪辑要求并确认看完配方，剪辑/发布完成后在弹窗里标记为完成即可。
- 每个案例会生成 `00-兼职须知.txt`；每次交付的「发给兼职文案」内置固定发布说明，减少对接时反复解释发布时间、封面、话题、素材顺序和完成确认方式。
- 交付内容会生成 `05-素材顺序清单.txt`，按顺序说明微信要发送的图片或视频。
- 视频/口播交付内容会额外生成 `03-口播字幕文案.txt` 和带剪辑填空卡、固定剪辑配方的 `04-剪辑要求.txt`；素材只按网页「本次视频素材」发送，剪辑或发布完成后只在系统标记完成，不要求上传成片。
- 图片任务仍会在本地保存 `图片任务说明.txt` 和生成图片，但工作人员主要在网页内查看和下载。
- 爆款互动动作会展示作品链接、播放/点赞/评论和互动建议，工作人员只需安排助理号/阑尾号互动并标记已处理。
