# 素人种草运营工作台 v1

本地网页工作台，用于管理兼职账号、三类内容排期、候选稿、素材扫描、网页内微信交付、图片任务、账号数据监控和爆款提醒。

## 启动

最简单方式：双击项目根目录的：

```text
启动素人工作台.command
```

它会自动检查环境、安装依赖、构建前端、启动本地服务并打开浏览器。

命令行开发方式：

```bash
cd /Users/licc/Desktop/素人系统/app
npm install
npm run dev
```

- 前端：`http://127.0.0.1:5173`
- 后端/API：`http://127.0.0.1:5174`

桌面窗口方式：

```bash
cd /Users/licc/Desktop/素人系统/app
npm run client
```

生成 Mac `.app` 目录：

```bash
cd /Users/licc/Desktop/素人系统/app
npm run client:mac
npm run client:package:check
```

打包输出在 `app/release/`。客户端包不会包含真实 `.env`、数据库和素材。Mac `.app` 会用 Electron 内置 Node 启动本地后端并打开工作台窗口，不需要工作人员手动启动服务。`client:package:check` 会实际启动打包后的 `.app`，确认内置后端和独立工作数据目录可用。后续如果要做 Windows 分发，需要在 Windows 机器上跑对应打包验收。

生产构建后也可以只跑后端：

```bash
npm run build
npm run start
```

打开：`http://127.0.0.1:5174`

## 本地数据

系统会自动创建：

```text
/Users/licc/Desktop/素人系统/data/souren.sqlite
/Users/licc/Desktop/素人系统/素材库/真实案例/
/Users/licc/Desktop/素人系统/素材库/服务器案例库/
/Users/licc/Desktop/素人系统/素材库/通用素材/
```

新建案例后，每个案例会自动生成：

```text
00-原始素材/
01-已筛选素材/
02-生成补充/
03-交付给兼职/
04-发布回收/
case.json
```

网页端只保存结构化数据和本地路径，大图片、视频和交付文件都放在本地素材库；日常发送动作在网页内完成。

共享盘/服务器目录是工作人员放案例原始素材的入口。可以在案例登记时直接填写「共享原始素材路径」，也可以把 `SOUREN_CASE_LIBRARY_ROOT` 指到服务器上的“待分配案例根目录”，然后在案例库页点击「扫描服务器案例库」一键登记。登记后系统会绑定该目录、同步到本地案例目录，并记录来源路径。

医院素材、套图素材和备用素材这类跨案例可复用素材放进 `素材库/通用素材/` 或 `SOUREN_SHARED_MATERIAL_ROOT` 指向的目录，在系统配置页扫描。案例素材和通用素材分开管理，后续图片生成可以同时引用“案例人物/过程素材”和“医院/通用场景素材”。

素材扫描会自动按目录名和文件名分类：案例人物、案例过程、案例对比、案例场景、医院素材、套图素材、生活场景、文字卡等。图片任务会把案例人物/变化参考和医院/通用场景素材一起写进参考素材，后续接 Image2 时可用于人物与场景合成。

## 图片接口

`app/.env` 里预留：

```env
PORT=5174
SOUREN_HOST=127.0.0.1
SOUREN_ACCESS_CODE=
SOUREN_CASE_LIBRARY_ROOT=
SOUREN_SHARED_MATERIAL_ROOT=
IMAGE_API_KEY=
IMAGE_API_URL=
IMAGE_MODEL=image-v2
IMAGE_SIZE=1024x1024
```

当前 v1 会创建图片任务和提示词。`IMAGE_API_KEY` 或 `IMAGE_API_URL` 为空时，任务显示为“待接入图片接口”，不会影响其他流程；两者都填好后可在网页内生成图片、预览并下载。

如果工作人员在另一台电脑上使用，把主机 `app/.env` 改成：

```env
SOUREN_HOST=0.0.0.0
SOUREN_ACCESS_CODE=给工作人员的访问码
```

重启后到「系统配置」查看局域网网址。工作人员端只打开网页、输入访问码；密钥、数据库、素材库仍在主机本地。远端工作人员不会看到打开主机本地目录的按钮，日常只用查看任务、复制、下载和标记。
如果开放局域网但没有填写 `SOUREN_ACCESS_CODE`，系统会直接拒绝启动。

## 当前闭环

1. 工作人员打开「今日中控台」，只看今天该处理的账号和动作。
2. 新建案例并生成本地案例目录。
   - 新建时会随机生成人设，并自动生成近期排期。
   - 工作人员默认只需要填兼职微信昵称、抖音主页/作品链接、项目和共享原始素材路径；人设和抖音号可以后续编辑。
   - 也可以在案例库使用「批量导入」，一次录入多名兼职/账号，并自动生成排期。
3. 工作人员把挑好的素材放进共享路径；如果是服务器待分配案例库里的目录，可在案例库先「扫描服务器案例库」并登记。
4. 系统先点「同步共享素材」，再点击扫描素材。
5. 系统生成排期槽位，对当天或近期槽位生成 3 条候选稿。
6. 工作人员选择一条并锁定。
7. 生成微信交付内容。
   - 如果没有可用素材，任务会进入「素材阻塞」，不会误进入可交付。
   - 素材阻塞任务回到案例详情查看素材缺口，补齐后重新生成交付内容；本机管理端才显示打开素材目录入口。
   - 如果是视频内容，可以从排期或案例详情创建「剪辑任务」，系统会生成本地剪辑任务单。
8. 工作人员点「查看交付内容」，在网页内复制文案、预览/下载图片或视频，用电脑微信发给兼职。
9. 标记派发；确认兼职或剪辑人员完成发布/交付后标记完成。
10. 账号数据由系统按活跃度登记采集任务；正在投放或起量的账号勤采，偏冷账号降频，休眠账号一周一次。主机端用已登录抖音的 Chrome 查看到期主页并把数据写入后台后，系统自动统计初始粉丝、最新粉丝和作品播放/点赞/评论。
11. 作品达到爆款阈值后，首页提醒安排助理号/阑尾号去评论区互动。
12. 工作人员在「爆款链接」里只粘贴抖音爆款作品链接；链接进入待分析状态，完成内容提取和结构拆解后再生成同类型内容。

系统会用账号采集数据反向调整排期：连续低播放账号自动降为两天一条或暂停自动补新排期，避免工作人员每天被平账号任务压着走。

如果账号已派发超过 3 天还没有确认完成，系统会按「失联暂停」处理：普通待发、补素材、剪辑和采集队列都会撤下，只在首页给一次「失联处理」动作。确认暂停后，这个号不会继续自动补排期或进入采集清单。

后台默认每 6 小时检查一次是否有账号到期；这只是检查间隔，不代表所有账号都会进入采集清单。

日常操作手册见项目根目录：`日常运营SOP.md`。

## 验收

```bash
cd /Users/licc/Desktop/素人系统/app
npm run doctor
npm run check
npm run client:check
npm run test:e2e
npm run verify
npm run verify:github
```

`test:e2e` 会临时创建测试数据库和素材目录，跑完后自动清理。
`verify` 会顺序运行 doctor、build、E2E，并额外启动隔离服务检查 `/api/health` 和 `/api/readiness`。
`verify:github` 会把远端主分支是否与本地提交同步作为硬性验收；如果本机没有写权限，它会失败并提示需要认证后上传。

## 新增实用操作

- 「排期规划」页可以按未来 7/14/30/60 天、状态、内容类型和账号关键词查看全局排期，并直接处理生成、交付、派发和完成确认。
- 「今日中控台」只保留一张「今日操作队列」，把微信交付、生成交付、选候选、生成候选、素材同步、爆款互动和剪辑任务按优先级放在同一个入口里。
- 「今日账号概览」只按兼职/账号汇总状态，不重复展开同一条任务的操作按钮。
- 「系统配置」页只显示工作人员能理解和使用的状态：局域网访问、素材目录、图片接口、内容阶段比例和素材模板。
- 「今日中控台」不把账号采集这类后台动作放到首屏；采集入口保留在底部折叠的后台状态里。
- 「今日中控台」会显示待同步共享素材；共享目录有新文件但还没同步到本地案例目录时，可直接在首页一键同步。
- 「今日中控台」支持一键准备今日交付：生成候选、锁定稳妥稿、生成交付内容；缺素材的任务会进入素材阻塞，不会计入可交付。
- 案例库支持批量导入兼职/账号，适合一次录入 40+ 账号。
- 案例库支持扫描服务器案例库，把共享盘/服务器里的待分配案例目录一键登记成案例，同时同步素材并生成近期排期。
- 案例库支持按关键词和健康状态搜索筛选，也可以删除误建案例；删除时本地案例素材目录会移入 `素材库/真实案例/_deleted/`，避免误删永久丢素材。
- 案例详情里本机管理端可以直接「打开素材目录」；局域网工作人员只在网页内复制说明、查看交付内容和标记状态。
- 案例详情支持「共享原始素材路径」和「同步共享素材」；共享盘/服务器只作为素材入口，系统生成交付使用本地受控副本。
- 系统配置页支持扫描和管理通用素材库，医院素材、套图素材和备用素材可以跨案例复用；扫描后可以在网页内预览、下载，并修正分类、用途和可用状态。
- 素材清单可以手动标记用途，例如案例人物、案例过程、案例对比、案例场景和日常素材，修正自动分类偏差。
- 案例详情里可以编辑案例信息；临时内容槽位由首页策略动作或后台维护生成。
- 案例详情会按项目模板显示素材缺口和账号健康提示。
- 异常账号会显示运营建议，把断更、素材缺口、播放下滑和爆款提醒转成下一步动作；未采集状态只放在后台采集区。
- 「账号策略动作」会按优先级汇总爆款互动、偏冷补内容和增长承接；偏冷和增长动作可以直接生成下一条排期槽位。
- 交付内容生成后，可以在排期卡片里直接「查看交付内容」，在网页内复制文案、预览/下载图片或视频。
- 候选稿卡片只负责选择和锁定内容；文案复制集中在「查看交付内容」弹窗里，避免重复入口。
- 首页「今日操作队列」只完整显示第一条任务，后面的任务只汇总数量和类型；队列清空后显示「今天发完了」。
- 排期卡片可以标记异常，避免卡在无法推进的内容上。
- 素材扫描后，可以把素材标记为「可用 / 需处理 / 不可用」。
- 素材缺口旁可以直接创建对应图片任务；补素材动作集中在案例详情里处理。
- 账号数据监控会记录初始粉丝、最新粉丝、作品播放/点赞/评论；默认使用 Chrome 登录态采集任务，主机端采集完成后写入后台。未采集状态不进入首页首屏和异常账号，不会伪造数据。
- 「爆款链接」页只要求工作人员粘贴抖音作品链接；系统先记录为待分析，首页也会显示待分析链接，完成内容提取和结构拆解后，再按适合/禁用人设关键词批量生成。
- 待爆款分析链接只保留打开原链接和补分析结果，避免把分析流程变成工作人员要复制的任务单。
- 图片任务在网页内展示提示词；接入图片接口后可以直接生成、预览、下载，并标记“待审核 / 已通过 / 已驳回”。
- 剪辑任务会在网页内展示固定剪辑配方；剪辑/发布完成后在首页把任务标记为完成即可。
- 每个案例会生成 `00-兼职须知.txt`；每次交付的「发给兼职文案」内置固定发布说明，减少对接时反复解释发布时间、封面、素材顺序和发完回传方式。
- 交付内容会生成 `05-素材顺序清单.txt`，按顺序说明微信要发送的图片或视频。
- 视频/口播交付内容会额外生成 `03-口播字幕文案.txt`、带固定剪辑配方的 `04-剪辑要求.txt` 和 `07-成片回收说明.txt`。
- 图片任务仍会在本地保存 `图片任务说明.txt` 和生成图片，但工作人员主要在网页内查看和下载。
- 爆款互动动作会展示作品链接、播放/点赞/评论和互动建议，工作人员只需安排助理号/阑尾号互动并标记已处理。
