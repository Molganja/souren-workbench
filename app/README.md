# 素人种草运营工作台 v1

本地网页工作台，用于管理兼职账号、三类内容排期、候选稿、素材扫描、网页内微信交付、图片任务和抖音核对队列。

## 启动

最简单方式：双击项目根目录的：

```text
启动素人工作台.command
```

它会自动检查环境、安装依赖、构建前端、启动本地服务并打开浏览器。

命令行开发方式：

```bash
cd /Users/licc/Desktop/素人系统/app
npm install
npm run dev
```

- 前端：`http://127.0.0.1:5173`
- 后端/API：`http://127.0.0.1:5174`

桌面窗口方式：

```bash
cd /Users/licc/Desktop/素人系统/app
npm run client
```

生成 Mac `.app` 目录：

```bash
cd /Users/licc/Desktop/素人系统/app
npm run client:mac
```

打包输出在 `app/release/`。客户端包不会包含真实 `.env`、数据库和素材。Mac `.app` 会用 Electron 内置 Node 启动本地后端并打开工作台窗口，不需要工作人员手动启动服务。后续如果要做 Windows 分发，需要在 Windows 机器上跑对应打包验收。

生产构建后也可以只跑后端：

```bash
npm run build
npm run start
```

打开：`http://127.0.0.1:5174`

## 图片 / 文案密钥

`app/.env` 里预留：

```env
PORT=5174
SOUREN_HOST=127.0.0.1
SOUREN_ACCESS_CODE=
IMAGE_API_KEY=
IMAGE_API_URL=
IMAGE_MODEL=image-v2
IMAGE_SIZE=1024x1024
LLM_API_KEY=
LLM_BASE_URL=https://dashscope.aliyuncs.com/compatible-mode/v1
LLM_MODEL=deepseek-v4-flash
DASHSCOPE_API_KEY=
LOCAL_CLAUDE_COMMAND=
SOUREN_LOCAL_AI_DISABLED=0
```

当前 v1 会创建图片任务和提示词。`IMAGE_API_KEY` 或 `IMAGE_API_URL` 为空时，任务显示为“待接入图片接口”，不会影响其他流程；两者都填好后可在网页内生成图片、预览并下载。

候选稿生成支持 OpenAI 兼容文案模型。填 `LLM_API_KEY` 或 `DASHSCOPE_API_KEY` 后，系统会优先用模型生成 3 条候选；模型未接入、超时或输出不合格时，自动回到本地模板生成。

本地助手默认自动查找 `claude`、`clude`、`claude-code`。如果命令不在 PATH 里，可以填写 `LOCAL_CLAUDE_COMMAND`，例如：

```env
LOCAL_CLAUDE_COMMAND=/opt/homebrew/bin/claude
```

如果只想生成助手工作包、不想调用本地命令：

```env
SOUREN_LOCAL_AI_DISABLED=1
```

如果工作人员在另一台电脑上使用，把主机 `app/.env` 改成：

```env
SOUREN_HOST=0.0.0.0
SOUREN_ACCESS_CODE=给工作人员的访问码
```

重启后到「系统配置」复制局域网网址。工作人员端只打开网页、输入访问码；密钥、数据库、素材库仍在主机本地。

## 当前闭环

1. 工作人员打开「今日中控台」，只看今天该处理的账号和动作。
2. 新建案例并生成本地案例目录。
   - 新建时会随机生成人设，并自动生成近期排期。
   - 工作人员默认只需要填对接咨询/负责人和抖音主页/作品链接；昵称、人设、抖音号都可以后续编辑。
   - 也可以在案例库使用「批量导入」，一次录入多名兼职/账号，并自动生成排期。
3. 把原始素材放进 `00-原始素材/`，点击扫描素材。
4. 系统生成排期槽位，对当天或近期槽位生成 3 条候选稿。
5. 工作人员选择一条并锁定。
6. 生成微信交付内容。
   - 如果没有可用素材，任务会进入「素材阻塞」，不会误进入可交付。
   - 素材阻塞任务可以直接复制补素材说明、打开素材目录，补齐后重新生成交付内容。
   - 如果是视频内容，可以从排期或案例详情创建「剪辑任务」，系统会生成本地剪辑任务单。
7. 工作人员点「查看交付内容」，在网页内复制文案、预览/下载图片或视频，用电脑微信发给兼职。
8. 标记派发、兼职已汇报。
9. 打开抖音链接核对并回填状态。
10. 工作人员在「爆款链接」里只粘贴抖音爆款作品链接；青豆提取、拆结构和生成同类型内容由助手处理。

日常操作手册见项目根目录：`日常运营SOP.md`。

## 验收

```bash
cd /Users/licc/Desktop/素人系统/app
npm run doctor
npm run consult
npm run check
npm run client:check
npm run test:e2e
npm run verify
npm run verify:consult
npm run verify:github
```

`test:e2e` 会临时创建测试数据库和素材目录，跑完后自动清理。
`consult` 会调用当前运行中的服务；如果服务没开，会临时启动服务，生成助手工作包并尝试调用本地助手。
`verify` 会顺序运行 doctor、build、E2E，并额外启动隔离服务检查 `/api/health` 和 `/api/readiness`。
`verify:consult` 会在上述验收后尝试调用本地助手，把当前工作包和关键本地文件路径发给 `claude`/`clude`/`LOCAL_CLAUDE_COMMAND`，要求它优先读取这些文件再给建议，并保存建议记录。
`verify:github` 会把远端主分支是否与本地提交同步作为硬性验收；如果本机没有写权限，它会失败并提示需要认证后上传。

## 新增实用操作

- 「排期规划」页可以按未来 7/14/30/60 天、状态、内容类型和账号关键词查看全局排期，并直接处理生成、交付、派发和核对动作。
- 「今日中控台」会按对接咨询/负责人汇总今日任务，工作人员能直接知道每条内容该发给哪个兼职、由谁对接。
- 「今日链路复盘」按状态串起待生成、待选择、已锁定、素材阻塞、可交付、已派发、已汇报、已核对，并支持复制今日工作简报。
- 「复制助手工作包」会在 `data/operator-packets/` 生成 Markdown，并把内容复制到剪贴板，里面包含下一步优先级、审阅指令、今日链路、待青豆分析、图片/剪辑任务、异常账号和关键路径。
- 助手工作包会附带当前 git 分支、提交、工作树状态、最近提交和关键代码文件路径，方便本地助手审阅系统进度。
- 「请本地助手建议」会优先调用 `claude`、`clude`、`claude-code`，也可以通过 `LOCAL_CLAUDE_COMMAND` 指定命令；结果会保存到 `data/ai-consults/`，首页会展示最近记录并能打开原文件。如果命令未安装，系统会保存调用记录，不影响其他功能。
- 「系统配置」页可直接打开助手工作包目录和建议记录目录。
- 「系统配置」页可复制本地助手接入说明，包含 `.env` 配置、验证命令和工作目录。
- `npm run doctor`、`/api/health` 和 `/api/config` 会显示本地助手命令是否可用；未安装时只是提醒，不阻断系统运行。
- 「系统配置」页会显示验收清单，把核心链路、外部接口、本地助手和本地目录区分为已就绪/待接入/需处理。
- 验收清单会显示远端代码是否与本地同步；当前本机缺少写权限时会显示需处理。
- 「今日中控台」会直接显示素材阻塞、待爆款分析、图片任务、剪辑任务、待核对任务。
- 「今日中控台」支持一键准备今日交付：生成候选、锁定稳妥稿、生成交付内容；缺素材的任务会进入素材阻塞，不会计入可交付。
- 案例库支持批量导入兼职/账号，适合一次录入 40+ 账号。
- 案例库支持按关键词、阶段、健康状态搜索筛选，也可以删除误建案例；删除时会同步删除本地案例素材目录。
- 案例详情里可以直接「打开素材目录」。
- 案例详情里可以编辑案例信息，也可以手动新增临时内容槽位。
- 案例详情会按项目模板显示素材缺口和账号健康提示。
- 异常账号会显示运营建议，把断更、待核对、素材缺口、播放下滑转成下一步动作。
- 交付内容生成后，可以在排期卡片里直接「查看交付内容」，在网页内复制文案、预览/下载图片或视频。
- 候选稿卡片可直接复制执行说明和抖音发布文案，方便发微信。
- 排期卡片可以标记异常，避免卡在无法推进的内容上。
- 素材扫描后，可以把素材标记为「可用 / 需处理 / 不可用」。
- 素材缺口旁可以直接创建对应图片任务，也可以复制补素材说明发给素材整理人员。
- 核对队列里点「核对回填」可用表单录入粉丝、播放、点赞、评论，并标记核对通过、内容异常或未找到作品。
- 「爆款链接」页只要求工作人员粘贴抖音作品链接；系统先记录为待分析，首页也会显示待分析链接，后续由助手用青豆提取、拆结构，再按适合/禁用人设关键词批量生成。
- 待爆款分析链接可以复制青豆任务，任务文本包含链接、提取字段、回填标准和下一步。
- 图片任务可以复制提示词；接入图片接口后可以直接在网页内生成、预览、下载，并标记“待审核 / 已通过 / 已驳回”。
- 剪辑任务会生成 `剪辑任务单.txt`，可复制给剪辑人员；剪辑/发布完成后在首页把任务标记为完成即可。
- 交付内容会生成 `05-素材顺序清单.txt`，按顺序说明微信要发送的图片或视频。
- 视频/口播交付内容会额外生成 `03-口播字幕文案.txt`、`04-剪辑要求.txt` 和 `07-成片回收说明.txt`。
- 图片任务仍会在本地保存 `图片任务说明.txt` 和生成图片，但工作人员主要在网页内查看和下载。
- 待核对任务可以复制抖音核对清单，按发布状态、发布时间、标题正文、素材顺序、数据和评论区问题逐项检查。
