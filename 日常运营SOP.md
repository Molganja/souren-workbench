# 素人系统日常运营 SOP

这份 SOP 面向每天实际使用系统的工作人员，也给本地助手做交接参考。

## 1. 每天启动

1. 双击 `启动素人工作台.command`。
2. 打开 `http://127.0.0.1:5174/`。
3. 进入「系统配置」，先看「系统验收清单」：
   - `已就绪`：可以正常使用。
   - `待接入`：通常是图片接口、文案接口或本地助手，不影响排期、文案、交付、核对。
   - `需处理`：优先处理，例如远端代码未同步、目录异常。

如果要用桌面窗口版，在终端运行：

```bash
cd /Users/licc/Desktop/素人系统/app
npm run client
```

生成 Mac 客户端目录：

```bash
npm run client:mac
```

Mac 客户端会用 Electron 内置 Node 启动本地后端，不需要工作人员手动启动服务。普通运营不需要理解这些命令，日常只用双击启动或打开客户端窗口。

如果工作人员在另一台电脑上使用，主机先在 `app/.env` 设置 `SOUREN_HOST=0.0.0.0` 和 `SOUREN_ACCESS_CODE`，重启后从「系统配置」复制局域网网址发给工作人员。工作人员只打开网址、输入访问码，不需要安装客户端。

## 2. 工作人员每天只做的动作

1. 看「今日中控台」。
2. 对 `待生成` 点生成候选。
3. 对 `待选择` 选一条并锁定。
4. 对 `已锁定` 生成交付内容。
5. 对 `可交付` 点「查看交付内容」，在网页内复制文案、下载图片或视频，再通过电脑微信发给兼职。
6. 发完后标记 `已派发`。
7. 兼职回传作品链接后点 `兼职已汇报`。
8. 进入 `待抖音核对`，打开抖音链接，回填播放、点赞、评论、粉丝和异常备注。
9. 看到爆款内容时，只需要在「爆款链接」粘贴链接。

## 3. 素材处理

新建案例后，系统会显示本地案例目录。把原始素材放入：

```text
00-原始素材/
```

然后回到案例详情：

1. 点「扫描素材」。
2. 把可用素材标记为 `可用`。
3. 如果任务进入 `素材阻塞`，先复制补素材说明，补齐后重新生成交付内容。

大图片、大视频、交付文件仍放本地文件夹，数据库只保存路径和状态；日常操作以网页内查看、复制、下载为主。

## 4. 微信交付规则

发送顺序固定：

1. 在「查看交付内容」里复制“发给兼职文案”并发送。
2. 复制“抖音发布文案”并发送。
3. 按页面里的“素材发送顺序”下载/发送图片或视频。
4. 提醒兼职发布后回传作品链接或截图。

不要把本地交付文件压缩后发给兼职；兼职端只接收普通文字、图片、视频。

如果是视频内容，页面会显示口播/字幕文案、剪辑要求和源素材下载。工作人员剪辑或安排发布完成后，只需要把剪辑任务标记为完成。

## 5. 爆款链接和青豆

工作人员只负责粘贴链接。

助手后续处理：

1. 从首页「待爆款分析」复制青豆任务。
2. 用青豆提取标题、正文/口播、评论区信息和结构。
3. 回填到爆款模板。
4. 按账号人设批量生成同结构内容。

链接未分析前不能批量生成，这是正常状态。

## 6. 本地助手建议

首页有两个入口：

- `复制助手工作包`：生成 Markdown，并复制到剪贴板。
- `请本地助手建议`：生成工作包，并尝试调用本地 `claude`、`clude`、`claude-code` 或 `LOCAL_CLAUDE_COMMAND`。

命令行也可以直接执行：

```bash
cd /Users/licc/Desktop/素人系统/app
npm run consult
```

这个命令会调用当前运行中的服务；如果服务没开，会临时启动服务，生成工作包并尝试调用本地助手。

工作包保存到：

```text
data/operator-packets/
```

顾问记录保存到：

```text
data/ai-consults/
```

如果本机没有装本地助手命令，系统会保存 `unavailable` 记录，不影响运营。

## 7. 抖音核对边界

当前默认做法是人工核对：

1. 系统打开抖音主页或作品链接。
2. 工作人员看发布时间、标题/正文、素材是否匹配。
3. 工作人员回填播放、点赞、评论、粉丝和异常备注。

后续可以做浏览器协同增强：客户端或浏览器插件尝试读取当前页面上的公开数据，再让工作人员确认。但它不能作为唯一方案，因为登录状态、页面改版、权限限制都会让自动读取失效。读不到时，仍按人工回填完成。

## 8. 交付前验收

每次改完系统后，在终端运行：

```bash
cd /Users/licc/Desktop/素人系统/app
npm run verify
```

如果要同时尝试本地助手：

```bash
npm run verify:consult
```

如果要把远端主分支同步也作为硬性验收：

```bash
npm run verify:github
```

当前如果本机没有远端写权限，`verify:github` 会失败，这是预期提示。需要先完成认证或配置有写权限的 SSH key，再上传。

桌面客户端配置验收：

```bash
npm run client:check
```

## 9. 当前已知外部依赖

- 图片接口未接入：图片任务会显示“待接入图片接口”，不影响文案、排期、交付、核对。
- 本地助手未安装：助手工作包和建议记录仍可生成，但不会得到真实建议。
- 远端代码写权限未配置：本地系统可用，但远端主分支不会自动同步。
