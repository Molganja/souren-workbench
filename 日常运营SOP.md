# 素人系统日常运营 SOP

这份 SOP 面向每天实际使用系统的工作人员。

## 1. 每天启动

1. 双击 `启动素人工作台.command`。
2. 打开 `http://127.0.0.1:5174/`。
3. 进入「系统配置」，先看「系统验收清单」：
   - `已就绪`：可以正常使用。
   - `待接入`：通常是图片接口，不影响排期、文案、交付和 Chrome 登录态采集清单。
   - `需处理`：优先处理，例如访问码、目录或素材状态异常。

如果要用桌面窗口版，在终端运行：

```bash
cd /Users/licc/Desktop/素人系统/app
npm run client
```

生成 Mac 客户端目录：

```bash
npm run client:mac
```

Mac 客户端会用 Electron 内置 Node 启动本地后端，不需要工作人员手动启动服务。普通运营不需要理解这些命令，日常只用双击启动或打开客户端窗口。

如果工作人员在另一台电脑上使用，主机先在 `app/.env` 设置 `SOUREN_HOST=0.0.0.0` 和 `SOUREN_ACCESS_CODE`，重启后从「系统配置」复制局域网网址发给工作人员。工作人员只打开网址、输入访问码，不需要安装客户端。

## 2. 工作人员每天只做的动作

1. 看「今日中控台」里的「今日操作队列」，从上到下逐条处理；再看「今日账号概览」，确认今天涉及哪些兼职/账号。
2. 对 `待生成` 点生成候选。
3. 对 `待选择` 选一条并锁定。
4. 对 `已锁定` 生成交付内容。
5. 对 `可交付` 点「查看交付内容」，在网页内复制文案、下载图片或视频，再通过电脑微信发给兼职；需要逐个确认时，先复制这个账号的清单。
6. 发完后标记 `已派发`。
7. 确认兼职或剪辑人员完成发布/交付后，标记 `已完成`。
8. 如果首页出现爆款提醒，安排助理号/阑尾号去评论区互动，然后标记已安排互动。
9. 看到爆款内容时，只需要在「爆款链接」粘贴链接。

## 3. 素材处理

工作人员先把挑好的案例素材放到共享盘/服务器对应目录。登记案例有两种方式：

1. 如果这个目录已经在服务器待分配案例库里，进入「案例库」点「扫描服务器案例库」，找到未登记目录后点「登记案例」。
2. 如果是临时单个目录，新建案例时直接填写兼职微信昵称、抖音主页、项目和这个案例的「共享原始素材路径」。

无论哪种方式，系统本地都会保留一份受控副本。

```text
共享原始素材路径/挑好的图片或视频
```

然后回到案例详情：

1. 点「同步共享素材」。
2. 点「扫描素材」。
3. 把可用素材标记为 `可用`。
4. 如果任务进入 `素材阻塞`，打开案例查看素材缺口，补齐后重新生成交付内容。

医院素材、套图素材、备用素材这类所有案例都可能用的内容，放到「系统配置」里显示的通用素材库，再点「扫描通用素材」。扫描后在「通用素材管理」里检查分类、用途和可用状态。

扫描后先看素材用途是否正确。案例自己的素材一般标成 `案例人物`、`案例过程`、`案例对比` 或 `案例场景`；通用素材一般会自动分成 `医院素材`、`套图素材`、`生活场景`、`文字卡`。如果自动分类不准，案例素材在案例详情里改，通用素材在系统配置的「通用素材管理」里改。

做图片任务时，系统会自动带入案例人物/变化参考和医院/通用场景素材；后面接 Image2 时，这些就是合成图的参考输入。

大图片、大视频、交付文件仍放本地文件夹，数据库只保存路径和状态；日常操作以网页内查看、复制、下载为主。

## 4. 微信交付规则

发送顺序固定：

0. 先看弹窗顶部“当前只发给谁”，微信窗口也只停在这个兼职上。
1. 第一次给这个兼职发内容时，先复制并发送“兼职须知”。
2. 在「查看交付内容」里复制“发给兼职文案”并发送。
3. 复制“抖音发布文案”并发送。
4. 按页面里的“素材发送顺序”下载/发送图片或视频。
5. 确认兼职已经按要求发布/交付后，在系统标记完成。

不要把本地交付文件压缩后发给兼职；兼职端只接收普通文字、图片、视频。

如果是视频内容，页面会显示口播/字幕文案、固定剪辑配方和源素材下载。工作人员只按配方替换素材，剪辑或安排发布完成后，把剪辑任务标记为完成。

## 5. 爆款链接

工作人员只负责粘贴链接。

待分析链接处理流程：

1. 打开「爆款链接」里的原视频链接。
2. 提取标题、正文/口播、评论区信息和结构。
3. 回到系统补分析结果。
4. 按账号人设批量生成同结构内容。

链接未分析前不能批量生成，这是正常状态。

待分析爆款链接不会进入「今日操作队列」。分析完成后，只给可投放账号生成同款候选；暂停、休眠、不符合投放频率或已有同日任务的账号会自动跳过。

## 6. 抖音数据监控

工作人员不需要逐条手动录入播放、点赞、评论和粉丝。

1. 新建案例时填写抖音主页。
2. 系统按账号活跃度登记采集任务：正在投放或起量的账号勤采，偏冷账号降频，休眠账号一周一次。
3. 主机端用已登录抖音的 Chrome 按清单打开主页，再通过底部折叠的「后台采集状态」写回账号快照和作品快照。
4. 系统自动统计初始粉丝、最新粉丝、作品播放、点赞和评论。
5. 首页只展示工作人员需要处理的账号策略动作，例如爆款互动、偏冷补内容或增长承接；偏冷和增长动作可以直接生成下一条排期。
6. 如果作品达到爆款阈值，首页会提醒安排助理号/阑尾号互动。

当天还没采集时，状态只放在底部折叠的「后台采集状态」里；系统不会伪造数据，也不会把待采集当成工作人员首屏任务。

排期也按采集结果减量：连续低播放账号会自动改为两天一条，连续多条不起色时暂停自动补新排期；已经锁定、交付、派发或完成的任务不受影响。

如果某个账号已派发超过 3 天仍未确认完成，系统会按「失联暂停」处理：普通队列撤下，采集暂停，只在首页提示一次是否确认暂停。

如果兼职后来又回来配合，在案例详情点击「恢复账号」。系统会把旧的超时派发任务标为「已取消」，并补新的近期排期。

## 7. 交付前验收

每次改完系统后，在终端运行：

```bash
cd /Users/licc/Desktop/素人系统/app
npm run verify
```

如果要把远端主分支同步也作为硬性验收：

```bash
npm run verify:github
```

当前如果本机没有远端写权限，`verify:github` 会失败，这是预期提示。需要先完成认证或配置有写权限的 SSH key，再上传。

桌面客户端配置验收：

```bash
npm run client:check
```

## 8. 当前已知外部依赖

- 图片接口未接入：图片任务会显示“待接入图片接口”，不影响文案、排期、交付和账号监控。
- 远端代码写权限未配置：本地系统可用，但远端主分支不会自动同步。
